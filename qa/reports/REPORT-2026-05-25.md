## [BUG-001] 作業中に start ハンドラが再発火すると cycles が二重カウント
- severity: medium
- area: timer
- repro:
  1. スタート（cycles=1, 作業中）
  2. WORKING 中に start 相当を再発火（`#start-btn` は display:none だが `el.click()` で発火可）
  3. cycles が 1 → 2 に増える
- expected: WORKING 中の start 再発火では cycles は増えない（INITIAL からのみ作業開始）
- actual: `status=WORKING` 再代入 → observer が「直前が POSE でない」と判定し `countupCycles()` 実行 → 二重カウント
- evidence: QA 一時スペック（削除済み）。`PomodoroTimerController.js` start ハンドラ 707 行付近
- 仮説: start ハンドラを `if (status === STATUS_ENUM.INITIAL.rawValue)` で囲えば防げる
- status: open

## [BUG-002] 一時停止中にスキップを押しても無反応（ボタンは表示されているのに効かない）
- severity: medium
- area: timer / ux
- repro:
  1. スタート → 一時停止（status=一時停止中、skip ボタンは表示されている）
  2. スキップを押す
  3. status が「一時停止中」のまま変化しない
- expected: skip ボタンが見えている以上、押せば次フェーズへ進む（or 押せないなら隠す）
- actual: `timer()` の switch が `*_POSE`(21/31/41) を処理せず default へ落ち、status 不変。実測 before="一時停止中" after="一時停止中"
- evidence: QA 一時スペック（削除済み）。`buttonDisplayUpdate` は POSE でも skip を inline-block 表示
- 仮説: skip 側で POSE→実行中へ補正してから `timer()` を呼ぶ、または POSE ケースで skip を非表示に。方針は Q-003 で要判断
- status: open

## 要・認識合わせ（開発エージェントへの確認事項）

### [Q-001] 初期 `#status` 表示は「開始」が正？
- severity: low / needs-decision
- HTML 既定は「作業中」だが `main()` が「開始」で上書きし、実表示は「開始」（badge は「停止中」）。意図通りか。「開始」と「停止中」が同時表示される語彙の不一致は意図的か。

### [Q-002] 長時間休憩のタイミング & 型
- severity: low / needs-decision
- 実トレースは「4 サイクル目の作業終了直後に長休憩」(4,8,12…) で一般仕様と一致、妥当。cycles=0 の誤発火経路は無し（WORKING 突入で必ず countup 済み）。
- ただし設定フォーム送信時 `longBreakFrequency` に**文字列**が入る（702 行、現状フォーム disabled で未発火）。この順序で確定でよいか／フォーム有効化時に `Number()` 変換を入れる想定か。

### [Q-003] POSE 中の skip の扱い
- severity: low / needs-decision
- 「使える」べきか「ボタンを隠す」べきか（BUG-002 の修正方針）。

### [Q-004] MutationObserver の堅牢性
- severity: low〜medium / needs-decision
- 高速連続操作・reset 直後 start でも例外/console error は出ず、クラッシュ未再現（status 更新が常に全置換 `textContent=` で childList ミューテーションになるため安全）。
- latent リスク: (1) `removedNodes[0]` がオプショナルチェーン無し（791/801/810 行）、(2) observer 設定は `characterData/subtree:true` を付けつつハンドラは childList 以外を無視（監視範囲のズレ）。
- 提案（挙動不変・堅牢性向上）: `removedNodes[0]?.textContent` 化＋設定を childList のみに整理してよいか。

## 開発エージェント対応記録（2026-05-25）

判断軸: UX 体験の最適化。

- **BUG-001 → resolved**: `start` ハンドラに `if (status !== INITIAL) return;` を追加（`PomodoroTimerController.js`）。WORKING 中の再発火で cycles が増えないことを回帰テストで確認。
- **BUG-002 / Q-003 → resolved**: 一時停止中(WORKING/BREAKING/LONGBREAKING_POSE)で skip を `display:none` に。「見えるのに押せない」を排除し、一時停止中の操作を再開/リセットに限定。skip を機能させる案は現 observer の脆さ(Q-004)で回帰リスク高と判断し見送り（将来機能として保留）。
- **Q-001 → resolved**: INITIAL 表示を「開始」→「待機中」に統一（enum string・badge・`index.html` 初期値）。状態語彙の不一致と初期チラつきを解消。
- **Q-002 → wontfix-now**: 設定フォームは現状 disabled で当該 submit は未発火。`cycles % longBreakFrequency` は `%` が数値強制するため機能上無害。フォーム有効化時に `Number()` 変換を入れる前提で保留。
- **Q-004 → partially-resolved**: `removedNodes[0]?.textContent`（791/801/810 行）に optional chaining を付与（挙動不変の保険）。observer 設定(characterData/subtree)の整理は価値小・リスク有で見送り。

再検証: `npx playwright test` → **24 passed**（回帰テスト BUG-001/BUG-002 を追加）。
